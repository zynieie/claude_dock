# 坞内拖拽设计 (V2.8)

> 目标：卡片拖拽全程丝滑、无闪烁、无悬空、无跳变。
> 原则：每一帧只做一次 DOM 写操作；定位全程用 `position:fixed` + 直接 `left/top`；布局变更仅在松手瞬间发生。

---

## 阶段总览

| 阶段 | 名称 | 触发 | 时长 |
|------|------|------|------|
| A | **捕获** (Capture) | `pointerdown` | 单次 |
| B | **启动** (Begin) | 位移 > 6px 阈值 | 单次 |
| C | **跟手** (Follow) | `pointermove` | 持续 |
| D | **落点** (Settle) | `pointerup` | 单次 |
| E | **收场** (Cleanup) | 落点动画结束 | 单次 |

---

## 阶段 A — 捕获 (Capture)

**触发**：鼠标左键按下卡片

**动作**：
1. 记录 `hwnd`（目标 Claude 会话 ID）
2. 记录 `baseRect = el.getBoundingClientRect()`（此时卡片在正常流，坐标最准）
3. 记录 `offX = e.clientX - baseRect.left`（鼠标在卡片内的相对偏移）
4. 记录 `offY = e.clientY - baseRect.top`
5. `setPointerCapture`（保证移出坞后事件仍发给该卡片）

**禁止**：不设任何样式、不读 ResizeObserver、不做 DOM 改动。

---

## 阶段 B — 启动 (Begin)

**触发**：累计位移 > 6px

**动作**（按顺序严格执行）：
1. `ResizeObserver.disconnect` 该卡片的 title + txt（防拖拽中重测）
2. 创建坞内 **占位（placeholder）**：一个等高不可见卡片插在原位，其它卡片保持不动
3. 把被拖卡片 `el` 的样式钉死：
   - `el.style.width  = baseRect.width  + 'px'`
   - `el.style.height = baseRect.height + 'px'`
   - `el.style.left   = baseRect.left   + 'px'`
   - `el.style.top    = baseRect.top    + 'px'`
   - `el.style.opacity = '0.92'`
   - `el.style.transform = 'scale(1.02)'`
4. 加 `.ghost` 类（启用 `position:fixed` + 阴影 + 高亮描边）
5. 把 `el` 移到 `#drag-layer`（fixed 全屏层，避开 overflow 裁剪）
6. 发 `bridge.dragStart(hwnd)` 信号给 Python

**禁止**：此阶段后不再读 `getBoundingClientRect()` 或任何布局属性。

---

## 阶段 C — 跟手 (Follow)

**触发**：每帧 `pointermove`（通过 `requestAnimationFrame` 节流）

### 坞内模式（鼠标在 dock 矩形 ± 8px 内）
- `el.style.left = (clientX - offX) + 'px'`
- `el.style.top  = (clientY - offY) + 'px'`
- `el.style.opacity = '0.9'`
- `el.style.transform = 'scale(1.02)'`
- 计算并在坞内显示蓝色插入线

### 坞外模式（鼠标不在 dock 矩形内）
- **卡片本身仍跟手，只是视觉变化**：
  - `el.style.left = (clientX - offX) + 'px'`
  - `el.style.top  = (clientY - offY) + 'px'`
  - `el.style.opacity = '0.55'`
  - `el.style.transform = 'scale(0.95) rotateZ(4deg)'`
- **不** snap 回坞内——卡片全程跟着鼠标走
- 隐藏坞内插入线
- 仅首次切换时发 `bridge.dragOutside(hwnd)` 让 Python 进入跟手模式
- 从坞外拖回坞内时发 `bridge.dragReturn()` 退出跟手

**禁止**：每帧只写 4 个样式属性；**绝不**在跟手期间读布局或增删 DOM。

### 插入线算法

```
取坞内所有非 ghost、非 placeholder 的卡片
求它们各自垂直中点 mid = top + height/2
找到 |mid - clientY| 最小的那张 target
插入线放在 target 的上沿（clientY < mid）或下沿（clientY >= mid）
设置 dragging.insertBefore = target
```

## 阶段 D — 落点 (Settle)

### 情况 D1：坞内落点 + 真位移
- 隐藏插入线
- 150ms ease-out 动画把 `left/top` 从当前位置过渡到最终槽位坐标（**丝滑归位，无跳变**）
- 动画结束后：
  - 把 `el` 放回 `#cards`，插入 `dragging.insertBefore` 对应槽位
  - 清除所有行内样式（left/top/width/height/opacity/transform）
  - 移除 placeholder
  - 重编所有 badge 编号
  - 重连 `ResizeObserver`
- 发 `bridge.dragEnd(true)`

### 情况 D2：坞外落点 + 真位移
- 隐藏插入线
- 用 150ms 动画把**卡片 fade-out + 缩小至消失**
- 发 `bridge.dragDrop(hwnd, clientX, clientY)` 让 Python 把真实窗口移到落点
- 移除 `el`（卡片从坞内消失 → "变成了窗口"）
- 重编剩余编号
- 重连 ResizeObserver

### 情况 D3：无位移（纯点击）
- 仅触发正常点击跳转 `bridge.jump(hwnd)`

---

## 阶段 E — 收场 (Cleanup)

所有分支在动画结束后统一执行：
- `el.classList.remove('ghost')`
- 清空行内样式
- 确保 `el` 在 `#cards` 正确槽位
- 重连该卡片的 `ResizeObserver`
- 清空 `dragging` 状态

---

## 性能约束

| 项 | 要求 |
|----|------|
| 跟手期间 DOM 写 | 每帧 ≤ 4 个样式属性 |
| 跟手期间 DOM 读（触发 reflow） | **0** 次 |
| ResizeObserver | 拖拽期间断开，收场重连 |
| 动画 | 仅落点阶段一次 150ms ease-out；跟手期间无 transition |
| 闪烁 | 禁止 display/visibility 切换卡片 |
| placeholder | 阶段 B 创建到阶段 E 移除，不闪动 |

---

## 坞外卡片视觉规格

| 属性 | 坞内 | 坞外 |
|------|------|------|
| opacity | 0.90 | 0.55 |
| transform | `scale(1.02)` | `scale(0.95) rotateZ(4deg)` |
| 插入线 | 显示 | 隐藏 |
| Python 模式 | 空闲 | 跟手真实窗口 |

---

## Bridge 信号清单

| 信号 | 时机 | Python 动作 |
|------|------|-------------|
| `dragStart(hwnd)` | 阶段 B | 记录 hwnd，保存当前窗口矩形 |
| `dragOutside(hwnd)` | 阶段 C 首次出坞 | 启动 16ms 定时器 |
| `dragReturn()` | 阶段 C 坞外→坞内 | 停定时器 |
| `dragDrop(hwnd, x, y)` | 情况 D2 | 停定时器；窗口移到落点 |
| `dragEnd(inDock)` | 情况 D1/D3 | 清理状态 |
